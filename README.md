# Todo Buddy Template

Full-stack TypeScript monorepo with Hono RPC, React, and PostgreSQL.

## Quick Start

```bash
# 1. Name your project (replaces @todo-buddy scope, display name, db name)
./setup.sh my-app

# 2. Install dependencies
pnpm install

# 3. Add shadcn components
cd frontend && pnpm dlx shadcn@latest add --all && cd ..

# 4. Start Postgres and set up env files
docker compose up -d
cp backend/.env.example backend/.env
cp frontend/.env.example frontend/.env

# 5. Generate better-auth schema (needs DATABASE_URL from .env and Postgres running)
cd backend && pnpm dlx @better-auth/cli generate --output src/db/auth-schema.ts && cd ..

# 6. Uncomment the auth-schema export in backend/src/db/schema.ts

# 7. Push database schema (creates auth tables)
pnpm run db:push

# 8. Start dev servers
pnpm run dev
```

Backend runs on http://localhost:3000, frontend on http://localhost:5173.

## Tech Stack

- **Frontend**: Vite 6, React 19, React Router 7, React Query 5, Tailwind CSS 4, shadcn/ui
- **Backend**: Hono 4 with Hono RPC (end-to-end type-safe API calls)
- **Database**: PostgreSQL 16, Drizzle ORM, postgres.js
- **Auth**: better-auth with email/password
- **Validation**: Zod (shared schemas between frontend and backend)
- **Package Manager**: pnpm workspaces

## Key Patterns

### Hono RPC
The backend exports `AppType` from `src/index.ts`. The frontend imports it to create a fully typed API client — no manual API wrapper needed.

```typescript
// Frontend: fully typed, auto-completed
const res = await client.api.posts.$get();
const data = await res.json(); // typed as Post[]
```

### Shared Zod Schemas
Validation schemas in `shared/schemas/` are used by both the backend (via `zValidator`) and frontend (via `react-hook-form` + `@hookform/resolvers`).

### Auth
better-auth handles `/api/auth/*` routes. All other `/api/*` routes require a valid session. The auth middleware attaches `user` and `session` to the Hono context.

## Scripts

| Command | Description |
|---------|-------------|
| `pnpm run dev` | Start backend and frontend dev servers |
| `pnpm run typecheck` | Type-check all packages |
| `pnpm run test` | Run all tests |
| `pnpm run db:push` | Push schema changes to database |
| `pnpm run db:generate` | Generate Drizzle migrations |
| `pnpm run db:seed` | Seed database with sample data |

## Regenerating better-auth Schema

The auth tables (`backend/src/db/auth-schema.ts`) are generated by the better-auth CLI. Re-run this after upgrading the `better-auth` package:

```bash
cd backend && pnpm dlx @better-auth/cli generate --output src/db/auth-schema.ts
pnpm run db:push
```
